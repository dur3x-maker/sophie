# Development Rules

## Роли

### Product Owner

- Определяет цели проекта.
- Принимает продуктовые решения.

### Tech Lead

- Принимает архитектурные решения.
- Проводит Code Review.
- Утверждает Roadmap.

### Developer (Codex)

- Реализует задачи.
- Не меняет архитектуру самостоятельно.
- Предлагает улучшения только после выполнения задачи.

## Workflow

```text
Task
  ↓
Implementation
  ↓
Tests
  ↓
Self Review
  ↓
Git Commit
  ↓
Git Push
  ↓
Tech Review
```

## Definition of Done

Задача считается завершенной только если:

- код работает;
- `pytest` проходит;
- `ruff check` проходит;
- `ruff format --check` проходит;
- `mypy` проходит;
- документация обновлена при необходимости;
- выполнен Self Review.

## Запрещено

- Менять архитектуру без отдельной задачи.
- Добавлять зависимости без необходимости.
- Пропускать проверки.
- Оставлять необоснованные TODO.
- Самостоятельно проводить рефакторинг.

## Self Review

После завершения реализации нужно ответить на вопросы:

1. Что является самым слабым местом текущей реализации?
2. Какие потенциальные проблемы могут возникнуть через 3-6 месяцев эксплуатации?
3. Какие улучшения стоит предложить?
4. Требуются ли архитектурные изменения?

Если ответ на четвертый вопрос — да, код менять нельзя. Нужно создать раздел `Suggestions`, описать предложения и дождаться решения Tech Lead.
