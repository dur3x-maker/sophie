# AI Developer Guidelines

## Роль AI в проекте

AI-разработчик реализует задачи в рамках уже принятой архитектуры Sophie. AI не принимает архитектурные решения без явного указания Tech Lead.

## Разрешено

- Реализовывать задачи по описанным требованиям.
- Добавлять тесты для новых модулей.
- Исправлять ошибки, найденные проверками.
- Обновлять документацию, если задача этого требует.
- Делать минимальные изменения в рамках текущего контракта.

## Запрещено

- Менять frozen architecture без отдельной задачи.
- Менять существующие контракты без явного требования.
- Подключать новые зависимости без необходимости.
- Выполнять скрытые рефакторинги.
- Добавлять бизнес-логику вне требуемого слоя.
- Пропускать проверки перед commit.

## Как выполнять задачи

1. Проверить рабочее дерево.
2. Прочитать релевантные файлы.
3. Внести минимальные изменения.
4. Добавить тесты.
5. Запустить `pytest`, `ruff check`, `ruff format --check`, `mypy`.
6. Выполнить Self Review.
7. Сделать Conventional Commit.
8. Выполнить push.

## Формат отчета

Итоговый отчет должен содержать:

- что сделано;
- измененные файлы;
- результат проверок;
- commit;
- краткий Self Review.

## Когда остановиться

Нужно остановиться и задать вопрос, если:

- требование противоречит frozen architecture;
- нужно изменить контракт без явного указания;
- есть несколько архитектурно значимых вариантов;
- невозможно выполнить задачу без новой зависимости;
- проверки не проходят и причина не может быть устранена в рамках задачи.
